Does Python Have Any Limitations?

The short and simple of it is YES, Python has many MANY limitating factors.

No language is perfect, and Python is no exception to this rule. There are many factors that programmers must consider before deciding which language they want to develop with.

The following videos go much more in depth than I can here. The FIRST video was created by prominent Social Media programming educator CS Dojo, and the SECOND video is a direct source from Google, one of Python's main backers.



Pros

Noob Oriented!

									print("hello, world!")

High Scalability

    Scalaility:

      The measurement of a system's ability to increase or decrease in peformance cost in response to changes in application and system processing demands

    Python, due to it's focus on simplicity, already comes pre-packaged with Data Structures, HUNDREDS of libraries created by the community, and a focus on readability/ease-of-use. These factors combine to turn Python into one of the most efficient languages available today.

    All of these features build onto one another to make Python as simplistic as possible for the coder, meaning that writing dynamic systems and using pre-packaged libraries to complete high-scaled tasks is relatively easy for an experienced developer.

Portability

    Python is a "WRITE ONCE, RUN ANYWHERE" programming langauge, this means that code written in Python on a PC can run on mobile phones, and vice versa, assuming of course you install the appropriate libraries and are using a Python Interpreter.

    Not only is it cross-paltform, but Python is even more effective running accross different OS. This means that Linux, Windows, and Unix-based Systems, can all run the same piece of Python code without any alterations.

Cons

Speed

    Python is an interpreted language, this means that the code is read line-by-line and translated to the machine by an interpreter. This contrasts heavily with Compiled languages, in which the code translates directly into Machine Code.

    While this difference may seem miniscule, it can become the difference between whether or not your program times out the server entirely. This means that Python coders once they become advanced enough must take the time to learn how to write their code more efficiently.



Too Dynamic

    Dynamic Typing

      The ability of a compiler or interpreter to assign a type to all declared variables at run-time.

    While commonly regarded as one of it's strongest features, Python's Dynamic Typing is the source of many developers frustration.

    The larger a project becomes, the more variables and complex algorithims are required to keep it running. This is less of an issue with static typing where variable types are declared as they're written, but when the Python interpreter dynamically changes the types of different variables at runtime, the system may throw massive errors that the programmer won't be able to correct without a careful, line-by-line search.



Memory Performance

    Memory

      In computing, memory is a device or system that is used to store information for immediate use in a computer or related computer hardware and digital electronic devices.

    Python has one major issue when it comes to it's memory usage and allocation:

    Garbage Collection

    Garbage Collection

      In computer science, garbage collection is a form of automatic memory management. The garbage collector attempts to reclaim memory which was allocated by the program, but is no longer referenced; such memory is called garbage.

    When you declare a variable in Python, that variable is not stored into your memory automatically. Instead, you must declare the type of variable you are storing, the system will then allocate the correct amount of memory to that variable's storage.

    The garbage collection is automated in Python, meaning that the system must actively keep a count of how many times a variable is referenced by an object, then subtracts by each time the variable is used. When that counter hits "0" the system will automatically deallocate the memory to free up space. This is called reference counting.

    What's The Issue With Reference Counting?

    Because Python is a single threaded language by design, a Global Interpreter Lock must be used to protect the reference count. If ran without an implemented GIL, errors could arise from situations Python is not built to handle, such as two threads increasing and decreasing the reference count simultaneously.

    This means that Python cannot be utilized at the maximum potential of systems that support multi-thread processing, and it is by designed built for simplistic, line-by-line execution at runtime. This essentially clogs memory allocation at larger scales, as the system can only use one of it's cores to process the reference count, and therefore variables aren't sent to garabge collection as qucikly as they otherwise might have been in a multi-thread enviornment.
