The Story of Python

    The history of Python can be broken down into a few major stages:

      Development

      Photo by David Clode on Unsplash

      Conceived in the late 1980s as a succesor to the ABC Language by Guido van Rossum, Python was a language designed for simplicity and quick-implementation times.

      Created and published by Tim Peters in 1999, the Zen of Python document lists FIVE core prinicples to the philosphy of Python:

      1. Beautiful is better than ugly

      2. Explicit is better than implicit

      3. Simple is better than complex

      4. Complex is better than complicated

      5. Readibility counts

Implementation and Maintennace

    In December of 1989, Python's implementation into various company systems and was later on released onto open-source websites such as Github. Five years later, in 1994, the main forumn for Python-related discussion, known as comp.lang.python, was created.

    The main feature that attracts people to Python is it's community, of which all members consistently work together to create libraries and features that would sometimes be added by van Rossum to the lanuage itself.

    Having served as the head developer and BDFL of Python for well over 30 years, van Rossum stepped down from his title on July 12th, 2018. The community reacted quickly to this surpsiing announcement, democratically electing a Python Steering Council that consists of FIVE members. The members currently serving on this community-organized board are:

    • Barry Warsaw
    • Brett Cannon
    • Carol Willing
    • Guido van Rossum
    • Nick Coghlan

    All of these people hold meetings to discuss issues in the community, and will have the final say on adding any new features or implementing fixes to problems brought up in Python forums.

Culture

    Image provided by u/TheRidley - r/ProgrammerHumor

    The culture of Python is by far it's biggest strength. Through internet memes and platforms such as Github, Reddit, Discord, and Stack Overflow, Python has cultivated a following of professional and noob programmers alike. The simplicity of Python helps to keep the community fresh, with new users constantly pouring in trying to learn their first programming language.

    There are two major types of Python users, the majority who use Python 3.0(Also known as Python 3000 or Py3k), and the smaller but vocal portion that use Python 2.x.

    These factions were created due to the major relase of Python 3.0 on December 3rd, 2008. This release served to massively cull duplicate constructs, modules, and functions, having been noted as a return to the Zen of Python.

    Due to the massive amounts of reworking that needed to be done to simplify this version of Python, backwards-compatibility with older versions was broken entirely. this means that entire programs running off of Python 2.x would no longer function if upgraded to Python 3.0

    By that point in time tens of thousands of programs were running off of Python! Meaning that, for a large majority of programmers, upgrading became much more of a pain than it was seemingly worth. Python 3.0 didn't offer them any new features, quite the opposite in fact, most libraries written for Python 2.x were never upgraded by their authors, meaning that a lot of people had no choice but to simply keep their older programs in the legacy version.
